The PhET Interactive Simulations project is actively engaged in efforts to increase the accessibility of interactive simulations for diverse students, including students with disabilities. As part of this work we have developed prototypes of PhET simulations with preliminary implementations of accessibility features. As you explore these prototypes, please make note of the specific browser and assistive technology they were tested with.

Keyboard Navigation and Auditory Description Prototypes

PhET simulations are rendered using SVG or Canvas (with future WebGL support planned), which makes it impossible to use conventional DOM-based approaches for accessibility. Below are prototypes of simulations containing implementations of keyboard navigation (allowing interactions without the use of a mouse or touch interface) and auditory descriptions (for screen readers).
Note: These prototypes are known to have many issues and are not intended for broad usage.

Forces and Motion: Basics (Net Force Screen) - Custom Support

Rendered with SVG. Uses global key listeners to provide custom keyboard accessibility support
Implementation: Focused elements are managed using custom data structures and the focus rectangle is drawn in SVG. Text is read aloud using a single ARIA live region, whose text is changed to respond to the users key events and to describe the initial scene.
Browser Compatibility: Even though it has only been tested on a few browsers, we expect this version to work well on many browsers.
Limitations: While it provides keyboard support, display of focus regions, and text readout, the elements are not exposed to assistive technology so some features (such as zooming) or changes based on user preferences/customizations are not available.
Link: 1.1.5-dev.4

Forces and Motion: Basics (Net Force Screen) - ARIA Support on SVG Elements

Rendered with SVG. Uses ARIA markup to present the elements to assistive technology
Implementation: Text is provided by "aria-label" and all elements are exposed as role="button"
Browser Compatibility: This version works on OS X Chrome + OS X Safari, and a nightly build of Firefox (does not work with Firefox 36.0.4). Windows + Chrome + NVDA (in forms mode) works ok. Other Windows browsers and screen readers have not been tested.
Limitations: This version exhibits significant differences on different browsers, but provides support for zooming and user preferences/customizations.
Link: 1.1.5-dev.9

Sonification Prototypes

Below are links to prototypes of PhET simulations that include sonification, i.e. the use of non-speech sounds to convey meaning.

Note: As of February 2015, these prototypes have only been tested in the Chrome browser.

Faraday's Law

Theremin-like sound with heavy reverb for voltage level and light bulb brightness, stereo panning that corresponds to needle deflection, various audio icons for controls. 1.0.1-sonification.9
Theremin-like sound with light reverb for voltage level and light bulb brightness, stereo panning that corresponds to needle deflection, various audio icons for controls. 1.0.1-sonification.8
Theremin-like sound with no reverb for voltage level and light bulb brightness, stereo panning that corresponds to needle deflection, various audio icons for controls. 1.0.1-sonification.7
Constant pitch sound for voltage level and light bulb brightness, uses different base pitch to distinguish between positive and negative voltage. Magnitude is indicated by volume of sound and low frequency modulation. Various audio icons for controls. 1.0.1-sonification.2

Concentration

Realistic sounds for water and solution being added and removed, fed through a filter in order to create a sonic cue for water depth. Realistic sound for shaker. Synthesized tones for solute selection. Various audio icons for controls. No sound for evaporation or concentration readout. 1.2.0-sonification.3
Simple sine wave tone for indicating water depth. Somewhat realistic sound for shaker. Various audio icons for controls. No sound for several major simulation features. 1.2.0-sonification.1
